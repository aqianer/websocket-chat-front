后端批量文件上传接口实现提示词
====================================

接口基本信息
------------
接口路径：POST /api/v1/documents/upload
请求格式：multipart/form-data
认证方式：Bearer Token（从请求头 Authorization 获取）


请求参数
--------

1. 文件参数
   参数名：files[]
   类型：文件数组
   说明：支持批量上传多个文件，文件字段名必须为 files[]（带方括号）
   文件类型限制：
     - PDF：.pdf
     - 文本文件：.txt, .md
     - Word文档：.doc, .docx
   文件大小限制：单个文件不超过 100MB
   文件数量限制：最多支持 300 个文件

2. 配置参数
   参数名：batchConfig
   类型：JSON 字符串
   说明：批量上传的配置信息，需要解析为 JSON 对象
   格式示例：
   {
     "splitStrategy": "auto",
     "language": "zh-CN",
     "knowledgeBaseId": 123
   }

3. 配置参数说明
   splitStrategy（分段策略）：
     - auto：自动分段与清洗
     - custom：自定义分段
     - hierarchy：按层级分段
   language（语言设置）：
     - zh-CN：简体中文
     - en-US：英语
     - 支持其他 ISO 639-1 语言代码
   knowledgeBaseId（知识库ID）：
     - 必填参数
     - 类型：整数
     - 说明：指定文档上传到的目标知识库ID
     - 示例：123

业务逻辑要求
------------

1. 文件验证
   - 验证用户 Token 有效性
   - 验证文件类型是否在允许范围内
   - 验证文件大小是否超过限制
   - 验证文件数量是否超过限制
   - 验证文件名是否合法（防止路径遍历攻击）
   - 验证知识库ID是否存在且当前用户有权限访问

2. 文件处理流程
   1. 接收并解析 files[] 文件数组
   2. 解析 batchConfig JSON 配置
   3. 验证知识库ID的有效性和权限
   4. 对每个文件进行验证
   5. 将文件保存到指定存储位置（本地文件系统或对象存储）
   6. 根据配置对文件进行预处理（分段、清洗等）
   7. 将文件信息与知识库关联保存到数据库
   8. 返回上传结果

3. 进度跟踪（可选）
   - 如果需要支持实时进度反馈，可以实现 WebSocket 或 Server-Sent Events
   - 返回每个文件的处理进度（0-100%）


技术栈建议
----------

Spring Boot 实现示例：

@RestController
@RequestMapping("/api/v1/documents")
public class DocumentUploadController {

    @PostMapping("/upload")
    public ResponseEntity<UploadResponse> uploadDocuments(
            @RequestParam("files[]") MultipartFile[] files,
            @RequestParam("batchConfig") String batchConfigStr,
            @RequestHeader("Authorization") String authHeader) {

        // 1. 验证 Token
        String token = authHeader.replace("Bearer ", "");
        if (!tokenService.validateToken(token)) {
            return ResponseEntity.status(401)
                .body(new UploadResponse(401, "未授权，请先登录", null));
        }

        // 2. 解析配置
        BatchConfig batchConfig = parseBatchConfig(batchConfigStr);

        // 3. 验证知识库ID
        validateKnowledgeBase(batchConfig.getKnowledgeBaseId());

        // 4. 验证文件
        validateFiles(files);

        // 5. 处理文件上传
        UploadResult result = documentService.uploadFiles(files, batchConfig);

        // 6. 返回结果
        return ResponseEntity.ok(new UploadResponse(200, "上传成功", result));
    }

    private void validateKnowledgeBase(Long knowledgeBaseId) {
        if (knowledgeBaseId == null) {
            throw new IllegalArgumentException("知识库ID不能为空");
        }

        // 验证知识库是否存在
        KnowledgeBase kb = knowledgeBaseService.getById(knowledgeBaseId);
        if (kb == null) {
            throw new IllegalArgumentException("知识库不存在");
        }

        // 验证当前用户是否有权限访问该知识库
        Long currentUserId = getCurrentUserId();
        if (!kb.getOwnerId().equals(currentUserId) && !hasAdminPermission(currentUserId)) {
            throw new IllegalArgumentException("无权限访问该知识库");
        }
    }

    private void validateFiles(MultipartFile[] files) {
        if (files == null || files.length == 0) {
            throw new IllegalArgumentException("请选择要上传的文件");
        }

        if (files.length > 300) {
            throw new IllegalArgumentException("最多只能上传 300 个文件");
        }

        for (MultipartFile file : files) {
            validateFile(file);
        }
    }

    private void validateFile(MultipartFile file) {
        String filename = file.getOriginalFilename();
        String extension = getFileExtension(filename);

        if (!ALLOWED_EXTENSIONS.contains(extension)) {
            throw new IllegalArgumentException(
                "不支持的文件类型：" + filename);
        }

        if (file.getSize() > 100 * 1024 * 1024) {
            throw new IllegalArgumentException(
                "文件大小超过限制：" + filename + "（最大100MB）");
        }
    }
}


数据库表设计建议
----------------

CREATE TABLE documents (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    file_name VARCHAR(255) NOT NULL,
    file_size BIGINT NOT NULL,
    file_path VARCHAR(500) NOT NULL,
    file_type VARCHAR(50) NOT NULL,
    upload_time DATETIME NOT NULL,
    update_time DATETIME NOT NULL,
    user_id BIGINT NOT NULL,
    knowledge_base_id BIGINT NOT NULL COMMENT '关联的知识库ID',
    split_strategy VARCHAR(50),
    language VARCHAR(10),
    status VARCHAR(20) DEFAULT 'pending',
    chunk_count INT DEFAULT 0,
    vector_status VARCHAR(20) DEFAULT 'not_vectorized',
    INDEX idx_user_id (user_id),
    INDEX idx_knowledge_base_id (knowledge_base_id),
    INDEX idx_upload_time (upload_time),
    INDEX idx_status (status),
    FOREIGN KEY (knowledge_base_id) REFERENCES knowledge_bases(id) ON DELETE CASCADE
);


安全注意事项
------------

1. 文件名安全处理
   - 防止路径遍历攻击（如 ../../../etc/passwd）
   - 使用 UUID 或时间戳重命名文件
   - 限制文件名长度

2. 文件内容验证
   - 验证文件的 MIME 类型与扩展名是否匹配
   - 扫描恶意文件（病毒、木马等）

3. 存储安全
   - 将上传文件存储在非 Web 可访问目录
   - 设置适当的文件权限
   - 定期清理临时文件

4. 速率限制
   - 实现上传速率限制，防止滥用
   - 限制单个用户的并发上传数


性能优化建议
------------

1. 异步处理
   - 文件上传后异步进行分段和向量化处理
   - 使用消息队列（如 RabbitMQ、Kafka）

2. 分片上传
   - 对于大文件，支持分片上传
   - 实现断点续传功能

3. 缓存优化
   - 缓存文件元数据
   - 使用 CDN 加速文件访问

4. 数据库优化
   - 批量插入文件记录
   - 使用索引优化查询性能

前端实现细节
------------

前端使用 Vue3 + Element Plus + Axios 实现，具体功能包括：

1. 路由参数传递
   - 路由路径：/document-upload-wizard/:kbId
   - 知识库ID通过路由参数传递（可选参数）
   - 前端在 onMounted 钩子中获取 kbId 参数
   - 如果没有提供 kbId，则显示警告并阻止上传

2. 文件选择与验证
   - 支持拖拽上传和点击上传
   - 文件类型验证：.pdf, .txt, .doc, .docx, .md
   - 文件大小验证：最大 100MB
   - 文件数量限制：最多 300 个

2. 上传进度显示
   - 每个文件独立显示上传进度（0-100%）
   - 使用 Element Plus 的 Progress 组件
   - 实时更新进度百分比

3. 文件状态管理
   - pending：待上传
   - uploading：上传中
   - success：上传成功
   - error：上传失败

4. 交互反馈
   - 上传中禁用删除和清空按钮
   - 上传成功显示"上传成功"标签
   - 上传失败显示错误信息
   - 按钮文本动态更新："上传中..."

5. 错误处理
   - 网络错误提示
   - 服务器错误提示
   - 文件验证失败提示
   - 支持重试上传

6. 路由跳转
   - 所有文件上传成功后自动跳转到"创建设置"页面
   - 步骤指示器更新

7. 用户界面
   - 文件列表展示（文件名、大小、进度、状态）
   - 清晰的视觉反馈（不同状态显示不同图标）
   - 响应式设计，支持移动端


注意事项
--------

1. 确保 batchConfig 参数正确解析为 JSON 对象
2. 文件字段名必须为 files[]（带方括号），否则前端无法正确发送
3. 返回的响应格式必须符合前端期望的结构
4. Token 验证必须在文件处理之前完成
5. 知识库ID是必填参数，必须验证其有效性和权限
6. 建议实现文件上传的断点续传功能以提升用户体验
7. 考虑实现文件上传的取消功能
8. 对于大文件上传，建议实现分片上传机制
9. 建议添加文件上传日志记录，便于问题排查
10. 确保数据库表中的 knowledge_base_id 字段有外键约束和索引
